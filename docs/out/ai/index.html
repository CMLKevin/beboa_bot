AI Systems Overview

Comprehensive overview of Beboa's AI personality, memory, and evaluation systems.

AI Systems Overview

Beboa's AI systems work together to create a dynamic, evolving personality that remembers users and adapts to interactions.

Architecture

Info

The LLM Evaluator sits at the center, processing all incoming interactions and routing them to the appropriate systems.

Core Systems

Personality

14 traits + 11 moods that evolve through interactions

User Memory

Per-user facts with semantic search and auto-extraction

Server Memory

Ambient awareness of all server conversations

Dynamic Personality

Beboa has 14 personality traits that evolve through interactions:

CategoryTraits
Big FiveOpenness, Conscientiousness, Extraversion, Agreeableness, Neuroticism
CustomPlayfulness, Sarcasm, Protectiveness, Curiosity, Stubbornness
HiddenAffection (Hidden), Mischief, Drama, Loyalty

Mood System

11 moods affect behavior in real-time:

  • happy - More playful, less snarky
  • soft - Rare genuine warmth
  • protective - Defensive, less snark
  • neutral - Default balanced state
  • mischievous - Scheming energy
  • curious - Engaged and questioning
  • annoyed - Short responses, extra sarcasm
  • flustered - Stuttering, aggressive deflection
  • dramatic - Overreacting to everything

Relationship Tracking

Each user has individual relationship metrics that grow over time:

MetricDescription
AffectionHow much Beboa likes them
TrustHow much she trusts them
FamiliarityHow well she knows them
RivalryCompetitive energy

6 Stages Stranger ‚Üí Acquaintance ‚Üí Regular ‚Üí Friend ‚Üí Close Friend ‚Üí Family

How They Work Together

1

Search Memories

User memory and server memory are searched for relevant context about the user and recent discussions.

1

Load Relationship

Relationship data is loaded to understand the user's history and trust level.

1

LLM Evaluation

The LLM Evaluator analyzes the message for mood impact and interaction quality.

1

Apply Personality

Personality traits and current mood shape the response style and tone.

1

Generate Response

Response generated with full context (personality + memories + server awareness).

1

Post-Process

Memory extraction stores new information; relationship updated based on interaction quality.

Meanwhile, in the background:

  • All messages are captured for server-wide memory (non-blocking)
  • Important messages (~30%) are queued for embedding
  • Hourly/daily summaries extract topics and trends

Configuration

All AI features can be configured via environment variables:

env
# Enable/disable systems
CHAT_ENABLED=true
MEMORY_ENABLED=true
MEMORY_AUTO_EXTRACT=true
SERVER_MEMORY_ENABLED=true
TOOLS_ENABLED=true
 
# LLM Evaluator
LLM_EVALUATOR_ENABLED=true
LLM_EVALUATOR_MODEL=x-ai/grok-4.1-fast
 
# Models
OPENROUTER_MODEL=x-ai/grok-4.1-fast
EMBEDDING_MODEL=openai/text-embedding-3-small

See Configuration for all options.
